Detection Query

The SPL

Identifies accounts requesting an unusual volume of TGS tickets using weak RC4 encryption, a hallmark of offline password cracking attempts against service principals.

kerberoasting_detection.spl 
index=windows sourcetype=WinEventLog:Security EventCode=4769
    earliest=-24h latest=now
    Ticket_Encryption_Type=0x17
    Service_Name!=krbtgt
    Service_Name!=*$
| bin _time span=1h
| stats
    count as tgs_requests
    dc(Service_Name) as unique_spns
    dc(dest) as unique_targets
    values(Service_Name) as targeted_spns
    by user src_ip _time
| where tgs_requests >= 5
| eventstats
    avg(tgs_requests) as avg_tgs
    stdev(tgs_requests) as stdev_tgs
    by user
| eval deviation_score = round((tgs_requests - avg_tgs)/stdev_tgs, 2)
| eval risk_score = case(
    tgs_requests >= 15 OR deviation_score >= 3, "CRITICAL",
    tgs_requests >= 10 OR deviation_score >= 2, "HIGH",
    tgs_requests >=  5, "MEDIUM"
)
| eval
    mitre_technique = "T1558.003",
    attack_name     = "Kerberoasting",
    encryption      = "RC4 (0x17)"
| sort - tgs_requests

Query Architecture

Detection Pipeline

Seven composable stages from raw event ingestion to risk-ranked output, each scoped to a single analytical responsibility.

1

Event Filtering

Aggregation

Compute TGS request volume, distinct SPN count, unique target systems, and SPN list per user/IP/time triplet.

aggregate
4

Threshold Pre-filter

Drop rows below 5 TGS requests to eliminate legitimate low-frequency access and focus on anomalous behaviour.

5

Statistical Baseline

Calculate per-user mean and standard deviation across the 24h window using eventstats without collapsing rows.

enrich
6

Deviation Scoring

Compute a z-score deviation metric per event — deviations ≥ 2σ or ≥ 3σ trigger escalated risk tiers.

score
7

MITRE Annotation & Sort

Enrich each result row with technique ID, attack name, and encryption label. Sort descending by request volume.
